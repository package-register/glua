# META-SOP：记忆管理规范

> 适用于 `.skcode/memory/` 目录下的技能/记忆文件

## 层级规则

| 层级 | 用途 | 更新频率 | 维护人 |
|------|------|----------|--------|
| L0 | META-SOP（本文） | 几乎不变 | 首次创建时确定 |
| L1 | 索引 + 技能总览 | 每次技能变更同步 | 执行者 |
| L2 | 独立技能文件 | 新技能晶体化时创建 | 执行者 |
| L3 | 引用文档/外部 SOP | 按需 | 执行者 |
| L4 | 原始会话数据 | 自动 | 系统 |

## 技能晶体化流程

1. **执行任务** → 解决具体问题
2. **提取模式** → 从执行路径中识别可重复的步骤
3. **写技能文件** → 按模板记录：目标 → 文件清单 → 关键代码 → 验证方法
4. **更新 L1 索引** → 在 gopher_lua_mem.md 中添加引用
5. **测试验证** → 确保技能描述的步骤仍然有效

## 文件规范

- `*.md` 文件使用 Markdown
- 技能文件以 `_skill_` 开头（如 `_skill_add_op.md`）
- 记忆文件以 `_mem_` 开头（如 `_mem_inventory.md`）
- 每文件尽量自包含，不超过 200 行

## 更新原则

- 只追加，不删除（除非信息错误）
- 技能文件不修改引用来源的原始代码
- 发现 bug 在技能中标注状态和修复版本
